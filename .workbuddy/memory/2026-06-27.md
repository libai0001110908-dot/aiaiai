## WorkBuddy skills 目录迁移到 D 盘

用户要求把本地 skills 目录从默认的 `C:\Users\cwl\.workbuddy\skills\` 改到 `D:\.workbuddy\skills\`。

### 调研结论
- WorkBuddy 的 `settings.json` 没有官方支持的自定义 skills 目录路径配置项
- CSDN 问答提到的 `customSkillsPath` 字段未经官方证实，盲目添加有风险
- skills 目录路径硬编码为 `~/.workbuddy/skills/`

### 执行方案：Windows Junction（目录联接）
1. 创建 `D:\.workbuddy\skills\` 目录
2. 复制原 `C:\Users\cwl\.workbuddy\skills\` 全部内容到 D 盘（含 7 个条目：install-portable-git-windows、excel-xlsx、self-improving-agent、summarize、word-docx 5 个 skill + 2 个 json 配置）
3. 重命名原目录为 `skills.bak` 作为备份
4. 用 PowerShell `New-Item -ItemType Junction` 创建 junction：`C:\Users\cwl\.workbuddy\skills` -> `D:\.workbuddy\skills`
5. 验证：通过 junction 可正常读取所有 skills 内容，数据实际存储在 D 盘（151K）

### 注意事项
- `cmd //c mklink /J` 在 Git Bash 中调用有编码问题，改用 PowerShell 工具成功
- junction 对 WorkBuddy 完全透明，应用仍访问原路径，实际数据在 D 盘
- `skills.bak` 备份保留，待用户重启 WorkBuddy 确认 skills 正常后可删除
- 操作需提权（dangerouslyDisableSandbox）因涉及 D 盘写入和系统级 junction 创建
